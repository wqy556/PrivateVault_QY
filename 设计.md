# PrivateVault 本地私密片库设计文档

## 1. 产品定位

PrivateVault 是一个无账号、无云端、无服务器的 Android 本地私密片库 App。

它不是普通相册，也不是网盘。核心目标是让用户按“片库 -> 影片 -> 详情图/链接/备注/标签”的方式管理本地私密资料。

所有数据默认只保存在本机。初版使用 Android App-specific storage，不内置 App 锁，不承诺文件级加密。

App 级加锁交给手机系统的应用锁、隐私锁或生物识别能力。PrivateVault 首页右上角只提供系统设置引导，不自行维护 PIN 或密保问题。

## 2. 核心对象

### 2.1 片库

片库是最大的分类。一个用户可以拥有多个片库。

片库支持：

1. 新增
2. 重命名
3. 删除
4. 查看片库内影片

不设置“待整理片库”这种系统默认分类。初始状态保留一个“我的片库”和一条“示例影片”，用于用户理解结构；不创建“收藏片库”。

### 2.2 影片

影片是片库下的一条记录，不等同于一个视频文件。

一个影片包含：

1. 片名
2. 多张详情图
3. 网盘地址
4. 备注
5. 自定义标签
6. 最近打开时间

片库和影片关系：

```text
片库 1 -> N 影片
```

### 2.3 详情图

每个影片可以对应多张详情图。

第一张详情图自动作为影片封面图，不单独维护封面字段。

添加详情图时，用户可以选择：

1. 复制：复制到 App 私有目录，系统相册原图保留，所以有两份。
2. 迁移/剪切：先复制到 App 私有目录，校验成功后，请求系统从相册移除原图。

只有原图从系统公共媒体库移除成功，才算迁移完成。否则状态应显示为“已复制，原图仍可能在相册可见”。

### 2.4 网盘链接

一个影片可以有多个链接。

每个链接包含：

1. 链接地址
2. 链接类型

链接类型包括：

1. 夸克
2. 百度网盘
3. 迅雷
4. 磁力
5. 网页
6. 其他

### 2.5 自定义标签

一个影片可以拥有多个标签，一个标签也可以属于多个影片。

关系：

```text
影片 N -> N 标签
```

## 3. 信息架构

底部导航：

1. 片库
2. 收藏
3. 设置

### 3.1 片库管理页

底部第一个“片库”Tab 直接进入片库管理页。

片库管理页展示：

1. 用户创建的多个片库
2. 新增片库入口
3. 重命名片库入口
4. 删除片库入口
5. 模糊搜索入口
6. 已有标签筛选入口
7. 底部最近打开，最多 5 条

点击片库进入片库详情页。

### 3.2 收藏页

底部中间 Tab 为“收藏”。

收藏页展示用户收藏的影片。

收藏页支持按片名模糊搜索，并通过系统已有标签筛选影片。筛选处只展示已有标签，不提供手动新建标签入口。

点击收藏影片进入影片详情页。

收藏页底部同样展示最近打开，最多 5 条。

### 3.3 片库详情页

片库详情页展示该片库下的影片列表。

每个影片一行，展示：

1. 片名
2. 封面图，即第一张详情图
3. 标签

影片条目不展示“几张图”这类统计，避免干扰列表扫描。

片库详情页支持按片名模糊搜索、按已有标签筛选，并提供“导出图片”功能。导出时会把该片库下所有影片的图片复制到系统相册，图集/文件夹名使用片库名称。视频导出后续单独处理。

点击影片进入影片详情页。

### 3.4 影片详情页

影片详情页展示：

1. 片名
2. 详情图列表
3. 添加详情图入口
4. 网盘地址列表
5. 每个网盘地址的链接类型
6. 备注
7. 自定义标签

### 3.5 设置页

设置页包含：

1. 系统应用锁说明
2. 是否允许截图
3. 存储占用
4. 隐私说明
5. 页面底部标注作者：秋漾

## 4. 返回规则

进入片库详情页、影片详情页、添加影片页等二级页面后，Android 系统返回键和右滑返回手势都应该返回上一层。

示例：

```text
片库管理 -> 点击片库 -> 片库详情 -> 右滑返回 -> 片库管理
片库管理 -> 点击片库 -> 片库详情 -> 点击影片 -> 影片详情 -> 右滑返回 -> 片库详情
```

## 5. MVP 范围

### 5.1 第一轮必须实现

1. App 启动后默认不锁定
2. 右上角打开锁图标引导用户去系统应用设置中启用应用锁
3. 第一屏展示片库管理
4. 片库管理页展示多个片库
5. 片库新增、重命名、删除
6. 中间 Tab 展示收藏影片
7. 点击片库进入影片列表
8. 影片列表一行一个影片，只展示第一张详情图作为封面、标题和标签
9. 点击影片进入影片详情
10. 影片详情展示详情图、链接、备注、标签
11. 添加详情图时区分复制和迁移/剪切
12. 片库页、收藏页、片库详情页支持模糊搜索和已有标签筛选
13. 片库详情支持导出该片库图片到系统相册对应图集
14. 系统返回键和右滑返回上一层

## 6. 数据对象

### 6.1 libraries

```sql
CREATE TABLE libraries (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

### 6.2 movies

```sql
CREATE TABLE movies (
    id TEXT PRIMARY KEY,
    library_id TEXT NOT NULL,
    title TEXT NOT NULL,
    notes TEXT,
    last_opened_at INTEGER,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY(library_id) REFERENCES libraries(id)
);
```

### 6.3 movie_images

```sql
CREATE TABLE movie_images (
    id TEXT PRIMARY KEY,
    movie_id TEXT NOT NULL,
    private_path TEXT NOT NULL,
    original_uri TEXT,
    import_mode TEXT NOT NULL,
    original_removed INTEGER NOT NULL DEFAULT 0,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL,
    FOREIGN KEY(movie_id) REFERENCES movies(id)
);
```

### 6.4 movie_links

```sql
CREATE TABLE movie_links (
    id TEXT PRIMARY KEY,
    movie_id TEXT NOT NULL,
    url TEXT NOT NULL,
    type TEXT NOT NULL,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL,
    FOREIGN KEY(movie_id) REFERENCES movies(id)
);
```

### 6.5 tags 与 movie_tags

```sql
CREATE TABLE tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    color TEXT,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

CREATE TABLE movie_tags (
    movie_id TEXT NOT NULL,
    tag_id TEXT NOT NULL,
    PRIMARY KEY (movie_id, tag_id),
    FOREIGN KEY(movie_id) REFERENCES movies(id),
    FOREIGN KEY(tag_id) REFERENCES tags(id)
);
```

## 9. 当前交互落地规则

1. 片库管理页是首页，负责展示和管理用户的多个片库。
2. 新增片库通过弹窗输入片库名称，空名称不可提交。
3. 重命名片库通过弹窗编辑原片库名称，保存后立即回到片库管理页。
4. 删除片库必须二次确认；删除片库时同步移除该片库下的影片记录。
5. 片库管理页和收藏页底部都展示“最近打开”，最多 5 条。
6. 收藏页只展示用户收藏的影片，不承担片库管理职责。
7. 进入片库详情或影片详情后，系统返回键和右滑返回应回到上一步。
8. 筛选区只能选择系统已有标签；新建标签仍在影片详情中完成。
9. 片库导出只导出图片，目标相册图集名使用片库名称。

## 10. 本轮功能实现边界

1. 数据持久化接入 Room，片库、影片、详情图、链接、标签关系按本地 SQLite 表保存。
2. 片库详情页支持通过片名创建影片，创建后影片会持久化到当前片库。
3. 影片详情页支持通过系统 Photo Picker 选择图片或视频。
4. 复制模式会把选中媒体复制到 App 私有目录，并保留系统相册原图。
5. 迁移/剪切模式会先复制到 App 私有目录，再通过 MediaStore 发起系统删除原图确认。
6. 如果系统删除确认被拒绝或无法创建删除请求，App 内仍保留已复制文件，但原图可能继续在相册、微信等系统媒体入口可见。
7. 迁移结果会保存到详情图记录中，影片详情页会显示原图是否已成功移除。
8. 新增片库、影片、详情图、链接和标签使用自增 ID（如 `library-1`），配合外键 CASCADE 保证引用完整性。
